# 项目文档

**更新规则**：任何功能、架构、写法更新，必须在工作结束后同步更新相关目录的子文档。

## 文档结构

```
docs/
├── README.md          # 本文档
├── ARCHITECTURE.md   # 系统架构总览
└── [模块名]/
    └── FOLDER.md     # 各模块文件夹的架构说明
```

## 如何使用

1. 遇到问题，先查看相关模块的 `FOLDER.md` 了解上下文
2. 修改任何文件后，同步更新头部注释和 `FOLDER.md`
3. 重大架构变更后，更新 `ARCHITECTURE.md`

> ⚠️ 本文档是自指文档体系的一部分。修改任何文件后，记得同步更新。
