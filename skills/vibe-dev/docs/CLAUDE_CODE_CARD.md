# Claude Code 快速参考卡

## 启动新项目

```
我要开始一个新项目：[描述]
请基于 vibe-dev 方法论：
1. 先帮我设计系统架构，输出到 docs/ARCHITECTURE.md
2. 建立分形文档结构
3. 再开始写代码
```

## 添加新功能

```
我要给 [模块] 添加 [功能]。
在写代码之前，请先生成技术方案（数据模型 + 接口 + 依赖关系）。
等我确认后再实现。
```

## 调试时（人类掌舵）

```
遇到了问题：[描述现象]
预期 vs 实际：[对比]
已尝试：[方案]

请先分析可能原因。我会告诉你根因，我们再修复。
```

## 完成后同步文档

```
验证通过。请同步更新：
1. [文件名] 的头部注释
2. docs/[文件夹]/FOLDER.md
3. docs/ARCHITECTURE.md（如有重大变更）
```

## 危险信号 - 立即停止

当出现以下情况，告诉 Claude Code 停止，主动要求定位根因：
- 同一个问题修改了 3 次还没解决
- 按了葫芦起了瓢
- 代码越改越乱

---

## 核心原则

1. **方案先行** → 不直接写代码，先出方案
2. **小步快跑** → 大模块拆成小任务，逐个验证
3. **文档即记忆** → 完成即同步，不留尾巴
4. **人类掌舵** → 根因在你手里，模型负责执行
